Hi! My name is Matt Rice and I am a Developer in Austin, Texas. I build websites, applications, and databases.

I primarily use Ruby and work within the Rails framework. Some of the main technologies I use are: PostgreSQL for my database; HTML, SCSS, ES6, jQuery, React, Flux, Redux, Ember for my front-end development; and RSpec, Cucumber for testing. I am very familiar using Git as my primary version control system, and I have launched most my apps on Heroku. In addition to Ruby, I have spent much time learning Golang and have built several command-line applications with it. I have also lightly played around with Python and R when I was branching out from my comfort zone in web development and experimenting with machine learning.

I have created over a dozen projects personally and collaborated with others on several more. I prefer collaborating with others but also work exceptionally well by myself. I am extremely dedicated to my craft and and passionate about growing and becoming a better developer.

If you would like grab a coffee and talk code, please feel free to contact me.

Slack-Roller

This app is developed to aid players, via Slack, in the game Dungeon World--a fantasy table-top role-playing games based on Dungeons and Dragons. This app takes in a request from a Slack channel via various commands and returns useful gameplay information and functions. Such commands include rolling various sided-dice; creating characters with stats, weapons, and armor; and attaching these characters and dice rolls to particular Slack users. Users can have multiple characters which they can switch between at any time--for example, if the user has multiple games with different characters for each.

ReRoot

ReRoot is an App geared towards allowing users to organize research in a tree-like structure. This allows users to better visualize their research progress and locate saved articles quickly. ReRoot is designed both for individual users and users who wish to work in teams on larger research projects. This project includes strict permissions dictating who can see what content; rerouting the user if he or she is unauthorized.

A large part of this project involved dealing with the speed issues associated with generating large amounts of data. Caching and eager loading proved effective but was incompletely implemented due to the recursive nature of the data. Nevertheless, most of the speed issues resulted from the large amount of nested partials and was resolved by converting some of these partials into React components.

Venter

Conway's Game of Life in Golang

This small project is a terminal application written in Golang with unit tests.

Rules: Every cell interacts with its eight neighbors, which are the cells that are horizontally, vertically, or diagonally adjacent. At each step in time (generation), the following transitions occur:

  1. Any live cell with fewer than two live neighbors die, as if by isolation,
  2. Any live cell with two or three live neighbors lives on to the next generation,
  3. Any live cell with more than three live neighbors dies, as if by overcrowding, and
  4. Any dead cell with exactly three live neighbors becomes a live cell, as if by reproduction
